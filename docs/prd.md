# 产品需求文档 (PRD)

## ExtHelper - 浏览器扩展管理插件

---

## 1. 产品概述

### 1.1 产品背景

随着浏览器扩展生态的繁荣，用户安装的扩展数量越来越多。浏览器自带的扩展管理页面功能有限，无法满足以下需求：

- **快速查找** - 大量扩展时难以快速定位
- **分类管理** - 缺少灵活的分组管理功能
- **快捷操作** - 需要多次点击才能完成简单操作
- **自动化管理** - 无法根据条件自动启用/禁用扩展
- **问题排查** - 无法快速定位导致问题的扩展

### 1.2 产品目标

构建一个**简洁、高效、跨浏览器**的扩展管理插件，帮助用户：

1. 快速管理和切换扩展状态（点击图标/开关）
2. 通过分组整理扩展（水平标签+弹窗）
3. 通过规则自动化管理扩展（域名/时间条件触发）
4. 通过二分法快速定位问题扩展（Bisect 调试）
5. 支持撤销/重做操作
6. 提升日常工作效率

### 1.3 产品定位

| 维度 | 描述 |
|------|------|
| 产品类型 | 浏览器扩展插件 |
| 核心功能 | 扩展启用/禁用，分组管理，规则自动化，Bisect 调试 |
| 目标用户 | 浏览器扩展高级用户、开发者 |
| 浏览器支持 | Chrome、Firefox、Edge |

## 2. 用户故事

### 2.1 核心用户故事

```
作为用户，我想要：
1. 在一个界面查看所有已安装的扩展
2. 点击开关/图标快速启用或禁用扩展
3. 通过分组标签快速管理某类扩展
4. 通过搜索和筛选快速找到目标扩展
5. 通过右键菜单进行更多操作（详情、设置、卸载）
6. 创建自动化规则，根据访问的网站或时间自动管理扩展
7. 使用 Bisect 功能快速定位导致问题的扩展
8. 撤销/重做扩展状态变更
```

## 3. 功能需求

### 3.1 核心功能 (P0)

#### F1: 扩展列表展示

```
功能描述：展示用户安装的所有浏览器扩展

需求详情：
- 支持三种视图模式：
  - 紧凑模式(Compact/Grid)：多列网格（CSS auto-fill），只显示图标和名称
  - 卡片模式(Card/List)：单列列表，显示图标、名称、描述、版本、开关
  - 详情模式(Detail)：单列列表，显示完整信息（权限、安装类型、操作按钮）
- 状态指示点显示启用/禁用状态

交互：
- 紧凑模式：点击卡片 → 切换启用/禁用状态
- 卡片/详情模式：点击开关 → 切换启用/禁用状态
- 右键点击 → 显示操作菜单
```

#### F2: 启用/禁用扩展

```
功能描述：快速切换扩展的启用状态

需求详情：
- 乐观更新，立即响应
- 状态持久化到浏览器
- 支持批量操作（启用全部/禁用全部）
- 支持撤销/重做

交互：
- 点击开关/图标 → 立即切换 → 视觉反馈
- 失败时自动回滚
```

#### F3: 搜索和筛选

```
功能描述：通过名称和描述搜索扩展，通过状态筛选

需求详情：
- 实时搜索，毫秒级响应
- 搜索扩展名称和描述
- 清空搜索按钮
- 筛选下拉菜单集成在搜索栏左侧（ALL/ON/OFF）
- 无结果显示空状态
```

#### F4: 分组管理

```
功能描述：管理扩展分组

需求详情：
- 水平分组标签栏（GroupsBar + GroupChips）
- 颜色标记分组
- 电源按钮一键启用/禁用分组内所有扩展
- 点击标签打开分组弹窗

分组弹窗（GroupModal）功能：
- 编辑分组名称
- 上传自定义图标
- 搜索和筛选扩展
- 扩展分两区：IN SECTOR（组内）/ NOT IN SECTOR（组外）
- 点击扩展切换组内/组外
- 组内扩展可切换启用/禁用
- 删除分组（带确认弹窗）

交互：
- 点击分组标签 → 打开编辑弹窗
- 点击"+"标签 → 创建新分组
- 点击电源按钮 → 一键切换分组内所有扩展
```

#### F5: 视图切换

```
功能描述：切换扩展展示方式

需求详情：
- 紧凑模式(GRID)：多列网格（CSS grid auto-fill, minmax(80px,1fr)），图标+名称
- 卡片模式(CARD)：单列列表，图标+名称+描述+版本+开关
- 详情模式(DETAIL)：单列列表，完整信息含权限列表+操作按钮

交互：
- 点击 Header 按钮组切换（GRID/CARD/DETAIL 三选一）
- 状态持久化
```

### 3.2 增强功能 (P1)

#### F6: Tab 导航

```
功能描述：在扩展管理和规则管理之间切换

需求详情：
- EXTENSIONS 标签：扩展管理主界面
- RULES 标签：规则自动化管理界面
- 当前标签高亮显示
```

#### F7: 右键菜单

```
功能描述：扩展操作菜单

菜单项：
- 启用/禁用（ENABLE/DISABLE）
- 打开设置页面（OPTIONS，仅扩展有设置页时显示）
- 查看详情（DETAILS）
- 卸载扩展（REMOVE）
```

#### F8: 扩展详情弹窗

```
功能描述：查看扩展详细信息

需求详情：
- 显示扩展图标、名称、版本、安装类型
- 显示启用/禁用状态标签
- 显示主页链接和设置链接
- 显示扩展描述
- 显示完整权限列表
```

#### F9: 撤销/重做

```
功能描述：支持扩展状态的撤销和重做

需求详情：
- 基于快照的历史记录
- 通过 Actions 菜单操作 Undo/Redo
- 显示可撤销/可重做的步数
- Bisect 期间禁用撤销/重做
```

#### F10: Bisect 调试

```
功能描述：二分法定位问题扩展

需求详情：
- 将启用的扩展分成两半
- 用户标记 Good（问题消失）或 Bad（问题仍在）
- 逐步缩小范围直到定位问题扩展
- 通过 Actions 菜单或 BisectBanner 操作
- 会话持久化，支持弹窗关闭后恢复
- 完成后可恢复原始状态

交互：
- Actions → Start Bisect → 开始二分
- BisectBanner 显示当前步骤、候选数量
- Good/Bad 按钮推进二分
- Cancel 取消并恢复
- Resolve 后显示嫌疑扩展，可恢复原始状态
```

#### F11: 规则自动化

```
功能描述：基于条件自动管理扩展

需求详情：
- 条件组（ConditionGroup）：域名匹配 + 可选时间表
  - 域名匹配模式：精确匹配、包含、通配符、正则表达式
  - 时间表：星期几 + 时间范围
- 动作：启用/禁用 扩展或分组
- 条件操作符：AND / OR
- 优先级排序
- 启用/禁用规则
- 复制规则
- 后台服务自动执行（监听标签页 URL 变化和定时器）

规则管理界面：
- 规则列表（支持搜索和筛选 ALL/ON/OFF）
- 规则编辑器弹窗
- 规则卡片显示条件摘要和动作摘要
```

#### F12: 批量操作

```
功能描述：批量管理扩展

菜单项（通过 Actions 下拉菜单）：
- Enable All - 启用所有扩展
- Disable All - 禁用所有扩展
- Start Bisect - 开始二分调试
- Undo / Redo - 撤销/重做
```

## 4. UI设计规范

### 4.1 页面布局

```
┌────────────────────────────────────────┐
│  Header (Logo + 视图切换 GRID/CARD/   │
│          DETAIL)                        │
├────────────────────────────────────────┤
│  TabBar ([EXTENSIONS] [RULES] [ACTIONS])│
├────────────────────────────────────────┤
│  SearchBar (筛选下拉 + $ 搜索框)      │
├────────────────────────────────────────┤
│  BisectBanner (仅 Bisect 激活时显示)  │
├────────────────────────────────────────┤
│  GroupsBar (水平分组标签 + 创建按钮)  │
├────────────────────────────────────────┤
│  Extension Grid/List (扩展卡片区域)    │
├────────────────────────────────────────┤
│  Footer (SYS_STATUS + LIVE指示器)      │
└────────────────────────────────────────┘
```

### 4.2 交互规范

| 操作 | 触发 | 结果 |
|------|------|------|
| 切换状态 | 点击开关（卡片/详情模式）/ 点击卡片（紧凑模式） | 启用/禁用切换 |
| 更多操作 | 右键菜单 | 显示操作列表（4项） |
| 查看分组 | 点击分组标签 | 打开分组编辑弹窗 |
| 启用/禁用全组 | 点击分组电源按钮 | 一键切换分组内所有扩展 |
| 关闭弹窗 | ESC/遮罩 | 关闭弹窗 |
| 切换视图 | Header按钮（GRID/CARD/DETAIL） | 三种视图模式切换 |
| 切换标签 | 点击 EXTENSIONS/RULES | 切换主内容区域 |
| 批量操作 | Actions 下拉菜单 | Enable All/Disable All/Bisect/Undo/Redo |

## 5. 非功能需求

### 5.1 性能需求

| 指标 | 目标 |
|------|------|
| Popup打开时间 | < 200ms |
| 扩展列表加载 | < 500ms |
| 搜索响应 | < 50ms |
| 状态切换 | < 100ms (乐观更新) |

### 5.2 兼容性需求

| 浏览器 | 最低版本 |
|--------|----------|
| Chrome | 120+ |
| Firefox | 121+ |
| Edge | 120+ |

### 5.3 安全需求

- 最小权限原则 (management, storage, tabs, alarms)
- Host permissions: `<all_urls>` (用于规则系统的域名检测)
- 本地存储优先
- 不收集用户数据

## 7. 开发模式

### 7.1 本地访问

Web 预览模式 (`pnpm dev:web`) 可通过 `http://localhost:4173` 访问页面。使用 `devStorage`（localStorage 后端）和 mock 数据模拟浏览器扩展 API。

扩展模式 (`pnpm dev`) 通过 Plasmo 框架加载为真实浏览器扩展。

### 7.2 Mock数据

包含20个模拟扩展、多个分组和50条预览规则，支持各种测试场景。
